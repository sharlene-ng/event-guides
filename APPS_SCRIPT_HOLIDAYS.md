# Apps Script changes for Public Holidays

The `/admin/holidays` page in the Next.js app talks to your Apps Script
backend via three new actions. Until you add these to the Apps Script
project, the holidays page will load fine but the list will be empty
and adding a holiday will return an error.

## 1. Add a "Holidays" tab to the spreadsheet

Create a new sheet tab called `Holidays` with this header row (row 1):

| id | date | name | createdAt |
|----|------|------|-----------|

- `id` — short random string, generated by the Apps Script
- `date` — the holiday date (YYYY-MM-DD)
- `name` — display name (e.g. "Hari Raya Aidilfitri")
- `createdAt` — ISO timestamp when added

## 2. Add the three actions to your Apps Script `Code.gs`

Add the snippet below alongside your existing `listEvents`/`createEvent`/
etc. handlers, then dispatch them from your `doGet`/`doPost` switch the
same way you dispatch the other actions.

```javascript
// ============ HOLIDAYS ============

function _holidaysSheet_() {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  let sh = ss.getSheetByName('Holidays');
  if (!sh) {
    sh = ss.insertSheet('Holidays');
    sh.getRange(1, 1, 1, 4).setValues([['id', 'date', 'name', 'createdAt']]);
  }
  return sh;
}

function listHolidays() {
  const sh = _holidaysSheet_();
  const last = sh.getLastRow();
  if (last < 2) return { ok: true, holidays: [] };
  const rows = sh.getRange(2, 1, last - 1, 4).getValues();
  const holidays = rows
    .filter(function (r) { return r[0]; })
    .map(function (r) {
      return {
        id: String(r[0]),
        date: r[1] instanceof Date
          ? Utilities.formatDate(r[1], 'Asia/Kuala_Lumpur', 'yyyy-MM-dd')
          : String(r[1] || ''),
        name: String(r[2] || ''),
      };
    });
  return { ok: true, holidays: holidays };
}

function createHoliday(payload) {
  if (!payload || !payload.date || !payload.name) {
    return { ok: false, error: 'Missing date or name' };
  }
  const sh = _holidaysSheet_();
  const id = Utilities.getUuid().slice(0, 8);
  const createdAt = new Date().toISOString();
  sh.appendRow([id, payload.date, payload.name, createdAt]);
  return {
    ok: true,
    holiday: { id: id, date: payload.date, name: payload.name },
  };
}

function deleteHoliday(payload) {
  if (!payload || !payload.id) {
    return { ok: false, error: 'Missing id' };
  }
  const sh = _holidaysSheet_();
  const last = sh.getLastRow();
  if (last < 2) return { ok: true };
  const ids = sh.getRange(2, 1, last - 1, 1).getValues();
  for (let i = 0; i < ids.length; i++) {
    if (String(ids[i][0]) === String(payload.id)) {
      sh.deleteRow(i + 2);
      return { ok: true };
    }
  }
  return { ok: false, error: 'Not found' };
}
```

## 3. Wire them into your action dispatcher

In your existing `doGet(e)` switch (where you handle `list`, `listContacts`,
`listPricing`, `listHallInfo`), add:

```javascript
case 'listHolidays':
  return _json_(listHolidays());
```

In your existing `doPost(e)` switch (where you handle `create`,
`updateStatus`, `uploadPoster`, etc.), add:

```javascript
case 'createHoliday':
  return _json_(createHoliday(payload));
case 'deleteHoliday':
  return _json_(deleteHoliday(payload));
```

(Use whatever helper names already exist in your script — `_json_` is just
the placeholder for "wrap in `ContentService.createTextOutput(...)`".)

## 4. Re-deploy the web app

After saving the script, redeploy the web app (Deploy → Manage deployments
→ edit → New version) so the new actions are exposed at the same URL.
